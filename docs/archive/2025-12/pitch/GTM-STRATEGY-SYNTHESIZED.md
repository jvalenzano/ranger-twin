## Content Strategy: The "Repurpose Ruthlessly" Model

You don't have time to create unique content for every channel. Here's the cascade:

```
ADR/Architecture Doc (already written)
    ↓
LinkedIn Post (300 words, key insight)
    ↓
Dev.to Article (expanded, code examples)
    ↓
Internal Lunch-and-Learn (slides + demo)
    ↓
GitHub README section (distilled)
```

**Example Flow:**
1. ADR-005 (Skills-First Architecture) already exists
2. → LinkedIn post: "Why we chose Skills over monolithic agents for federal AI"
3. → Dev.to: "Skills-First Architecture: A Pattern for Federal Multi-Agent Systems"
4. → Lunch-and-learn: Walk through ADR-005 with live demo
5. → GitHub README: "Architecture" section links to ADR

One piece of thinking, five pieces of content.

## What NOT To Do (Anti-Goals)

1. **Don't pitch adjacent agencies yet.** You need USFS proof first.
2. **Don't over-invest in GitHub community building.** Polish the repo, but don't expect viral adoption of a federal forestry tool.
3. **Don't create a content calendar you can't sustain.** 2 LinkedIn posts/week is enough. Quality > quantity.
4. **Don't position against Esri/Palantir.** You're not competing—you're complementing.
5. **Don't forget attribution.** "Leading RANGER development at jvalenzano" in every bio. They're paying for this work.
